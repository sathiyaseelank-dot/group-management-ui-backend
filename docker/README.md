# Docker Test Environment

Single-machine Docker setup for testing the full ZTNA tunnel flow with `ztna-client`.
Spins up two isolated networks, each with a connector + agent + nginx resource.
The controller and frontend stay on the host — not containerised.

```
Host (controller :8081/:8443, frontend :3001, ztna-client)
  │
  ├── ztna-net-1  (172.20.1.0/24)
  │     ├── connector-1   :9443 gRPC  → host port 19443
  │     │                 :9444 tunnel → host port 19444  ← ztna-client connects here
  │     ├── agent-1       (outbound only to connector-1:9443)
  │     └── resource-1    172.20.1.10:80  (nginx)
  │
  └── ztna-net-2  (172.20.2.0/24)
        ├── connector-2   :9443 → host 29443 | :9444 → host 29444
        ├── agent-2
        └── resource-2    172.20.2.10:80  (nginx)
```

## Quick Start

### 1. Build the Rust binaries

```bash
cd services/connector && cargo build --release && cd ../..
cd services/agent     && cargo build --release && cd ../..
cd services/ztna-client && cargo build --release && cd ../..
```

### 2. Ensure the controller is running

```bash
make dev-controller   # or: cd services/controller && go run .
```

### 3. Run the setup script

```bash
./scripts/docker-test-setup.sh
```

The script:
1. Sources `services/controller/.env` to get `JWT_SECRET` / `INTERNAL_CA_KEY`
2. Derives an admin JWT (no browser login needed)
3. Creates 2 remote networks, 2 connectors, 2 agents via the controller API
4. Issues 4 enrollment tokens
5. Writes env files to `docker/env/`
6. Builds and starts the Docker containers
7. Waits for all 4 components to enroll and come online
8. Creates 2 resources, 1 user group, 2 access rules
9. Adds the first admin user to the group
10. Prints test instructions

### 4. Test the tunnel

**SOCKS5 mode (no root required — easiest for bug testing):**

```bash
ZTNA_MODE=socks5 \
CONTROLLER_URL=http://localhost:8081 \
ZTNA_TENANT=<your-workspace-slug> \
SOCKS5_ADDR=127.0.0.1:1080 \
INTERNAL_CA_CERT="$(cat docker/ca.crt)" \
./services/ztna-client/target/release/ztna-client

# In another terminal — test both requests:
curl -x socks5h://127.0.0.1:1080 http://172.20.1.10/   # first request  → should work
curl -x socks5h://127.0.0.1:1080 http://172.20.1.10/   # second request → was the bug
curl -x socks5h://127.0.0.1:1080 http://172.20.2.10/   # different resource
```

**TUN mode (requires root):**

```bash
sudo CONTROLLER_URL=http://localhost:8081 \
ZTNA_TENANT=<your-workspace-slug> \
CA_CERT_PATH=docker/ca.crt \
./services/ztna-client/target/release/ztna-client

# In another terminal:
curl http://172.20.1.10/
curl http://172.20.1.10/   # second request
```

### 5. Tear down

```bash
./scripts/docker-test-teardown.sh

# Also remove controller DB objects:
CLEAN_CONTROLLER=1 ./scripts/docker-test-teardown.sh
```

## Files

| File | Purpose |
|------|---------|
| `docker-compose.test.yml` | Defines all containers and networks |
| `docker/Dockerfile.connector` | Connector container (binary bind-mounted from host) |
| `docker/Dockerfile.agent` | Agent container (binary bind-mounted from host) |
| `docker/entrypoint-connector.sh` | Enrolls on first start, then runs; skips if cert exists |
| `docker/entrypoint-agent.sh` | Runs `agent run` (agent re-enrolls on every start) |
| `docker/nginx/resource-{1,2}.html` | Unique HTML pages served by each resource |
| `docker/env/*.env` | Generated env files with enrollment tokens (gitignored) |
| `docker/ca.crt` | Extracted CA cert (gitignored, generated by setup script) |
| `scripts/docker-test-setup.sh` | Full automated setup |
| `scripts/docker-test-teardown.sh` | Clean teardown |

## Environment Variables for the Setup Script

| Variable | Default | Description |
|----------|---------|-------------|
| `CONTROLLER_URL` | `http://localhost:8081` | Controller HTTP URL |
| `CONTROLLER_ADDR` | `localhost:8443` | Controller gRPC address |
| `TRUST_DOMAIN` | `mycorp.internal` | SPIFFE trust domain |
| `ENROLL_WAIT_SEC` | `60` | Seconds to wait for components to enroll |

All other values are read from `services/controller/.env`.

## Troubleshooting

**Connector/agent fails to start:**
```bash
docker compose -f docker-compose.test.yml logs connector-1
docker compose -f docker-compose.test.yml logs agent-1
```

**Re-enroll from scratch** (clear connector state volumes):
```bash
./scripts/docker-test-teardown.sh
./scripts/docker-test-setup.sh
```

**Admin JWT rejected (HTTP 401/403):**
Verify `JWT_SECRET` or `INTERNAL_CA_KEY` in `services/controller/.env` matches the running controller.

**QUIC tunnel not working:** `9444/udp` must be open in your firewall:
```bash
sudo ufw allow 19444/udp
sudo ufw allow 29444/udp
```
QUIC failure is non-fatal — the client falls back to TLS automatically.

**nftables errors in agent logs:**
The agent container needs `CAP_NET_ADMIN`. This is already set in `docker-compose.test.yml`.
On some kernels, also add `--privileged` to the agent service for testing.
